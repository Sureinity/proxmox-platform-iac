# Documentation

This repository uses the Diataxis documentation model.

## Structure

- `tutorials/`: guided orientation for new engineers
- `how-to/`: task-focused instructions for common project activities
- `reference/`: factual project contracts, layouts, naming rules, and scope
- `explanation/`: architecture rationale and design decisions in prose
- `adr/`: accepted architecture decision records
- `roadmap/`: implementation phases and milestone summaries
